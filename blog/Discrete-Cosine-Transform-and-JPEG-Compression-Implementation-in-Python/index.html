Introduction: Why Care DCT?

We have been using Discrete Cosine Transform (DCT) without possibly awaring of it: DCT is actually the JPEG compression standard!

In audio analysis, DCT is used in computing MFCC coefficients to extract feature vector (for more on MFCCs, see [KC] in reference section), which makes deep learning on sound possible by

  • Treating the sequence of MFCC coefficients as a time sequence of data (at each "window" of suitable "hop length", we get a float vector of fixed size) or;

  • Stacking the MFCC coefficients and treat it as an image.

In the former case the use of sequence model such as LSTM or Transformer becomes possible. The later case makes it possible to use convolution layer such as Conv1D in Keras.

Basic Knowledge to Work With

Definitions

In what follows for i,j=0,1,2,…,N−1i,j=0,1,2,…,N−1 we will denote

uij=αjcos(2i+1)jπ2Nandαj={1√Nif j=0,√2Nif j≠0.
uij=αjcos(2i+1)jπ2Nandαj=⎧⎩⎨⎪⎪⎪⎪⎪⎪1N−−√2N−−−√if j=0,if j≠0.

For each fixed jj we will consider uj(x):=uxj=αjcos(2x+1)jπ2Nuj(x):=uxj=αjcos(2x+1)jπ2N as a function on a discrete domain. The vectors

{u:j:=[uij=αjcos(2i+1)jπ2N for 0≤i<N]∈RN|j=0,1,…,N−1}
{u:j:=[uij=αjcos(2i+1)jπ2N for 0≤i<N]∈RN∣∣∣j=0,1,…,N−1}

form an orthonormal basis in RNRN, by stacking all them together column by column

U:=[|||u:0u:1⋯u:N−1|||],
U:=⎡⎣⎢|u:0||u:1|⋯|u:N−1|⎤⎦⎥,

then UU is orthonormal: UTU=IUTU=I.

The orthogonality of UU and the construction of the basis u:ju:j will be explained in Mathematics behind DCT section. For now let's enjoy the coding and see the results.

Computations

We make explicit calculations in order to make readers comfortable with the definition above.

First, given a gray-scale image f:[0,N−1]×[0,N−1]→Rf:[0,N−1]×[0,N−1]→R there are always unique coefficients F:[0,N−1]×[0,N−1]→RF:[0,N−1]×[0,N−1]→R such that

f(p,q)=N−1∑i=0N−1∑j=0αiαjF(i,j)cos((2p+1)iπ2N)cos((2q+1)jπ2N).
f(p,q)=∑i=0N−1∑j=0N−1αiαjF(i,j)cos((2p+1)iπ2N)cos((2q+1)jπ2N).(∗)

In fact by using UU defined in the previous section. Denote also f,Ff,F as the matrix with entry f(p,q),F(p,q)f(p,q),F(p,q) at pp-th row and qq-th column, if we define F=UTfUF=UTfU, then

f=UFUT.
f=UFUT.

You can expand the RHS (UFUTUFUT) to convince yourself that eventually you get the same expression as in (∗)(∗). On the other hand, by using this UU we have the reverse:

F(x,y)=N−1∑h=0N−1∑k=0f(h,k)αxαycos((2h+1)xπ2N)cos((2k+1)yπ2N),
F(x,y)=∑h=0N−1∑k=0N−1f(h,k)αxαycos((2h+1)xπ2N)cos((2k+1)yπ2N),

as this is not a summation of coscos's of fixed frequencies, that's why we don't define f=UTfUf=UTfU.

Our coding will strictly follow the notation and calculation in this section.

Basis of Cosine Transform
def range2D(n):
  indexes = []
  for i in range(0, n):
    for j in range(0, n):
      indexes.append((i, j))
  return indexes

def alpha(p):
  return 1/np.sqrt(N) if p == 0 else np.sqrt(2/N)

def cos_basis(i,j):
  def cal_result(a):
    x=a[0]
    y=a[1]

    return alpha(i) * alpha(j) * np.cos((2*x+1) * i * np.pi/(2*N)) *  np.cos((2*y+1) * j * np.pi/(2*N))
  return cal_result

Compute Coefficients for Cosine Transform

Let's prepare the following 8×88×8 matrix that convert the original pixels into compacted energy distribution:

U = np.zeros((N,N))

for i in range(0, N):
  for j in range(0, N):
    U[i, j] = alpha(j) * np.cos((2*i+1)*j*np.pi/(2*N))

U_t =  U.transpose()

You may check that U is indeed orthonormal by:

Mask or Truncate the Energy Distribution

Here we use a naive approach, we screen out the fourier coefficents by simply retaining those from the upper-left corner. For that, we create a mask:

def create_mask(closure):
  mask = np.zeros((N, N))
  for i in range(N):
    for j in range(N):
      if closure(i,j):
        mask[i, j] = 1
  return mask

For example, by calling mask = create_mask(lambda i,j: 0<=i+j<=2) our mask will be like:

Result: Apply Cosine Transform
original = load_img(IMAGE_PATH, target_size=(512, 512), color_mode="grayscale")
original = img_to_array(original).astype("uint8")
original = np.squeeze(original)
original.shape

mask = create_mask(lambda i,j: 0<=i+j<=2)
print(mask)
steps = int(original.shape[0]/N) # we choose 512x512 image, therefore steps = 64
print(steps)
compressed = np.zeros_like(original)

for x in range(steps):
  for y in range(steps):
    sub_pixels = original[N * y: N*(y+1), N * x: N*(x+1)]
    sub_pixels = sub_pixels - 127.5
    fourier_coefficients = np.matmul(np.matmul(U_t, sub_pixels), U)
    fourier_coefficients = fourier_coefficients * mask
    reverted_pixels = np.matmul(np.matmul(U, fourier_coefficients), U_t)
    reverted_pixels = reverted_pixels + 127.5
    compressed[N * y: N*(y+1), N * x: N*(x+1)] = reverted_pixels

Mathematics behind DCT

Notation. Every vector will be considered as a column vector whenever the computation does not make sense when they are represented as a row.

From DFT

Let N≥1N≥1 be an integer and denote w=e2πi/Nw=e2πi/N, define vk:=(1,wk,w2k,…,w(N−1)k),vk:=(1,wk,w2k,…,w(N−1)k), then the matrix

V:=[|||v0v1⋯vN−1|||]
V:=⎡⎣⎢|v0||v1|⋯|vN−1|⎤⎦⎥

is orthogonal simply because their inner product

(vk,vℓ)=N−1∑j=0(wk)j(¯w)j=(wk¯wℓ)N−1wk¯wℓ−1⏟≠0 if k≠ℓ

is 0 when k≠ℓ, where wN=1. Therefore the basis V:={v0,v1,…,vN−1} forms an orthogonal basis in CN.

But how to find a similar basis in Rn? It is not as simple as taking the real part of the linear combination v=V[v]V=∑aivi since the coordinate [v]V∈CN even v∈RN.

DCT-2
Strategy of Construction

Consider the symmetric second difference matrix:

A=[⊗=1⊗′=−100⋯0−12−10⋯00−12−1⋯0⋮⋱⋮−12−10000⋯0⊠=−1⊠′=1]

which arises when a second derivate is approximated by the central second difference:

−f″(x)=1h2(−f(x−h)+2f(x)−f(x+h))+o(h)

as h→0.

Let uk,:∈RN denote the values of a function evaluated on the discretized domain [x0,x1,…,xN−1] of [0,π−1N], i.e., uk,:=fk(x:) for some fk:[0,π−1N]→R, where : means stacking values by running through all indexes at that position.

Our cosine transform is based on decomposing a function into a linear combination of cosine functions (of different frequencies) in discrete case. To find them, we will choose uk,: such that Auk,:=λkuk,:. By the fact that:

Fact. Eigenvectors associated to different eigenvalues are linearly independent.

we then obtain a set of eigenvectors (of different eigenvalues) solving the approximated problem

−1N2f″k(x:)≈Auk=λkfk(x:).

Which function would solve the ODE −f″=N2λf? Which is the trigonometric function!

Why (⊗,⊗′)=(1,−1) and (⊠,⊠′)=(−1,1)?

These two assignments are determined by imposing boundary conditions. More specific, we extend the domain from [0,π−1N] to [−1N,π], imagine we are solving ˜uk∈RN+2 (append one point at the beginning and the tail of x:∈RN respectively).

We have not imposed any boundary condition to our solution fk yet. We will require our function be symmetric at −12N (or at j=−12 if j=−1,0,1,…,N, xj=jπN), this implies fk(x−1)=fk(x0).

At another boundary due to the shift above, we try to require f′k(xN−12)=0, this implies fk(xN−1)=fk(xN).

{uk,−1=fk(x−1)=fk(x0)=uk,0uk,N−1=fk(xN−1)=fk(xN)=uk,N

and plug this condition into the second difference formula to get:

{−uk,−1+2uk,0−uk,1=uk,0−uk,1−uk,N−2+2uk,N−1−uk,N=−uk,N−2+uk,N−1.

These become the necessary condition and thus have determined our (⊗,⊗′)=(1,−1) and (⊠,⊠′)=(−1,1), and therefore ensure our solution is a cosine function.

Different values of ⊗'s and ⊠'s will correspond to different boundary condition imposed on fk for other combinations, see [GS] for a complete classification.

The solution corresponding to (⊗,⊗′)=(1,−1) and (⊠,⊠′)=(−1,1) is usally called the basis of DCT-2 (or simply DCT).

Derivation of the Basis of Discrete Cosine Transform

We have spent many effort to determine the matrix A to work with, let's start with computing the basis directly for our only candidate:

Let ℓ(x)=ax+b for some a,b∈R, denote

uk,0:N=(1,wℓ(1)k,wℓ(2)k,…wℓ(N−1)k),

note that we have ∑N−1k=0uk,:=∑N−1i=0(∑N−1k=0wℓ(i)k)ei=1−wNℓ(i)1−wℓ(i)=0 whenever ℓ(i)≠0 for integer i. As we will see this is indeed the case later for ℓ(x)=12x+14.

Since ℓ(j±1)=ℓ(j)±a, then for j=1,2,…,N−2, coordinate-wise:

[Auk,:]j=−wℓ(j−1)k+2wℓ(j)k−wℓ(j+1)k=(2−(wak+w−ak))wℓ(j)k=(2−2cos2akπN)uk,j.

Therefore Auk,1:N−1=(2−2cos2akπiN)uk,1:N−1 for whatever linear ℓ we choose (here xh:k=[xi for i s.t. h≤i<k]).

By taking real part on both sides, as A is real, we have

A[∑1≤j<N−1cos(2πNℓ(j)k)ej]=(2−2cos2akπN)∑1≤j<N−1cos(2πNℓ(j)k)ej,

here ej denotes the standard basis in RN. It remains to require uj:=cos2kπNℓ(j) satisfies u−1=u0 and uN−1=uN, from which we can determine a,b and thus ℓ.

The equation u1=u0 implies cos(2kπN(−a+b))=cos(2kπN(b)), since cos is even, it is sufficient to require

a−b=b⟺a=2b.

The equation uN−1=uN implies

cos(4kbπ−2kbπN)=cos(4kbπ+2kbπN),

for this to hold, it is sufficient to require 4b=1 (as cos is always symmetric about x=kπ for every k∈Z), altogether we have a=12, and thus for k=0,1,2,3,…,N−1,

2πNℓ(j)k=2πN(12j+14)k=(2j+1)kπ2N.

Finally, we have

zk:=∑0≤j<Nαjcos((2j+1)kπ2N)ej,Azk=(2−2coskπN)zk

for suitably chosen αj's, j=0,1,2,…,N−1.
